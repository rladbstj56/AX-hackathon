## 슬라이드 2 — 문제 이해 (0:30–1:30)

> 담당자 신뢰 확보 구간. 문제를 내 언어로 다시 정의.

**받은 문제 (원문 그대로)**
- 5개 채널·8주 성과 CSV로 매주·매월 "어느 채널이 효율적인가, 무엇이 변했나"를 보고. 그 과정이 수작업 반복.

**내가 재정의한 진짜 페인포인트**
- 표면 문제는 "시간이 오래 걸린다"지만, 진짜 문제는 **① 매번 사람이 계산해 실수가 섞이고 ② 결과가 '숫자 나열'로 끝나 정작 "그래서 뭘 해야 하나"가 빠진다**는 것.
- 누가·언제·얼마나: 마케팅 담당 DA가 **매주·매월 반복**, 회당 반나절 이상. *(과제 제공 시나리오 기준 — 실측 아닌 추정)*

**문제의 크기 (Before)**
| 지금 방식 | 비용 |
|---|---|
| 리포트 1회 작성 | 반나절 이상 (추정) |
| ROI·변화율·채널 비교 | 손계산 → 수식·셀 오류 상시 위험 |
| 산출물 | 수치 나열, 실행 안(案) 없음 |

**스코프 (정직하게 선 긋기)**
- **푼 것**: 정제 → 계산 → 이슈 탐지 → 진단 리포트 → 예산 재배분안, 새 CSV 자동 대응(재현성).
- **의도적으로 안 푼 것**: 실시간 대시보드/웹앱, 광고 플랫폼 API 자동 연동, 채널 간 교차 기여(MTA/MMM) 정밀 배분 — 3시간 스코프 밖이라 한계로 명시(슬라이드 6).

*발표 멘트*: "문제를 '리포트 빨리 만들기'가 아니라, '사람 손을 빼서 실수를 없애고, 숫자 뒤에 실행안을 붙이기'로 다시 정의했습니다."

## 슬라이드 3 — 해결 접근 & 의사결정 (1:30–2:30)

> 결과보다 재현 가능한 사고 과정이 채용 신호.

**접근 가설 — 왜 이렇게 풀었나**
- "정확성"과 "해석"은 잘하는 주체가 다르다. **계산은 실수 없는 Python에, 해석·서술은 규칙+Claude에, 반복 실행은 Skill에** 맡기는 3분할이 반나절 수작업을 명령 한 줄로 바꾸는 열쇠.

**버린 대안 + 버린 이유 (트레이드오프)**
| 고려한 선택지 | 접은 이유 |
|---|---|
| Claude가 직접 숫자 계산 | LLM 계산은 검증 불가·재현성 없음 → 계산은 Python으로 고정 |
| Streamlit 인터랙티브 웹앱 | 매주 '자동 생성'이 목표지 '보는 화면'이 목표가 아님 → 정적 리포트로 충분 |
| 매번 프롬프트로 리포트 재작성 | 사람 개입 = 매번 품질 편차 → Skill로 파이프라인 고정 |

**핵심 흐름 (비전공자용 흐름도)**
```
성과 CSV 넣기
   ↓  ① 데이터 청소 — 중복 지우고, 빈 칸·입력 오류 골라내기
   ↓  ② 정확히 계산 — 채널별 효율·전주 대비 변화를 Python이 계산
   ↓  ③ 이상 신호 탐지 — 돈이 새는 곳·급변한 곳을 자동으로 찾아냄
   ↓  ④ 리포트 작성 — "무슨 일이 있었나"(진단) + "예산 어떻게 옮길까"(처방)
결과물: 리포트 + 예산 재배분안 (매주 같은 품질로)
```

**★ 막혔던 지점 & 돌파 (실전 문제해결력)**
- **문제**: 처음엔 제공된 샘플 데이터에서만 리포트가 잘 나왔다. 그런데 "매주 다른 데이터로도 돌아가야 한다"가 핵심 목표.
- **발견**: 채널·주차를 줄인 가짜 데이터로 돌려보니 **"손실 이슈가 하나도 없는 주"에서 리포트 생성이 멈춰버림**(빈 결과를 처리 못 함). 샘플 데이터만 봤으면 절대 못 잡았을 함정.
- **돌파**: 이런 빈 경우(손실 0건, 오가닉 없음 등)를 안전하게 넘어가도록 고치고, **가짜 데이터 4종을 상시 자동 점검(회귀 테스트)으로 박제**. 이제 어떤 데이터를 넣어도 멈추지 않는지 명령 한 줄로 매번 검증된다.
- 의미: "한 번 되는 코드"가 아니라 **"매주 안 깨지고 도는 파이프라인"**으로 만든 지점.

**AI vs 나의 판단**
| AI/Python에 맡긴 것 | 내가 직접 결정한 것 |
|---|---|
| 사칙연산·이상치 탐지·엣지 처리 | 어떤 지표를 볼지 (전체효율 MER과 광고 순효율을 분리 표기) |
| 리포트 문구 규칙 채우기 | 이상치 판정 기준·급변 임계값을 어디에 둘지 |
| 반복 실행(Skill 발동) | 예산 재배분 우선순위 규칙 = **임팩트 × 빈도** (반복되는 낭비를 위로) |
- 이 판단들의 근거는 전부 `decisions.md`에 기록 → 나중에 왜 이렇게 했는지 재현·설명 가능.

*발표 멘트*: "가장 자신 있는 부분은 여기입니다. 샘플에선 안 보이던 함정을, 가짜 데이터로 일부러 부숴보며 찾아냈고, 다시는 안 깨지게 자동 점검으로 묶었습니다."

## 슬라이드 4 — 산출물 & 실증 (2:30–3:30, 발표 하이라이트)

> 담당자는 기술 이해도가 낮다. "어떻게 만들었나(코드·모델)"는 빼고 "무슨 일을 대신해 주는가"로.

**데모 영상** `[핵심 기능 실제 동작 장면 삽입 위치]`
- 실데이터로 한 사이클: CSV 지정 → 명령 한 줄 → 리포트·재배분안·HTML 대시보드 생성.

**실제 동작 1건 (입력 → 처리 → 출력)**
```
[입력]  8주치 성과 데이터 파일 (280줄, 채널 5개)
   ↓
[처리]  자동: 중복 1줄 제거 · 빈 칸 3건 표시 · 매출 입력오류 1건 격리
        채널별 효율 계산 · 돈 새는 곳 자동 탐지 · 예산 이동안 도출
   ↓
[출력]  ① 진단 리포트   — 핵심 수치 + 채널 ROI 순위 + 이슈 + 전주 대비 변화
        ② 예산 재배분안 — "메타광고에서 3,194,281원 빼서 네이버광고로"
        ③ HTML 대시보드 — 위 둘을 보기 좋게 (이미지 아닌 표·요약)
```

**핵심 결과물 스크린샷** `[insight_report.md / budget_reallocation.md / HTML 대시보드 화면 캡처 삽입]`

**이 도구가 대신해 주는 일 (기술 용어 없이)**
- "이번 주 어느 채널이 잘했고 못했나" → 채널별 순위표로 자동 정리
- "돈이 어디서 새고 있나" → 저효율 지출을 금액으로 찍어서 알려줌
- "그래서 뭘 하면 되나" → **"이 채널에서 저 채널로 얼마 옮기세요"**까지 제안
- 깊은 기술 구조(스크립트·계산식·규칙)는 전부 별첨으로.

*발표 멘트*: "보시는 것처럼, 사람이 반나절 하던 일을 명령 한 줄이 대신하고, 끝에 '예산을 이렇게 옮기라'는 실행안까지 붙습니다."

## 슬라이드 5 — 임팩트 (3:30–4:15)

> 모든 수치엔 근거·가정. 과장하지 않는 게 오히려 신뢰.

**Before / After 정량 표**

| 리포트 1회 작성 시간 | 반나절 이상 *(추정)* | 명령 한 줄 (초 단위 실행) |

- 저효율 지출 **8,822,424원**(메타광고 과집행, W5) 자동 탐지

- 매출 데이터 왜곡 **2건(합 30,223,909원)** 발굴 → '손실'과 분리해 "원본 재확인 대상"으로 정직하게 표기

**정성 효과**

*발표 멘트*: "숫자는 부풀리지 않았습니다. '반나절'은 추정이라 밝혔고, 순증 매출도 상한값이라 명시했습니다 — 그게 오히려 이 리포트가 믿을 만하다는 증거입니다."

## 슬라이드 6 — 적용 가능성 & 한계 & 확장 (4:15–4:45)

> 정직한 한계 + 구체적 확장 = 신뢰의 핵심. 완벽한 척이 오히려 감점.

**우리 회사에 들고 오면 (적용 조건)**
- 필요한 건 **같은 형태의 성과 CSV 하나**뿐(채널·기간·주차 라벨이 달라도 자동 적응).
- 별도 API 키·추가 비용 없이 Claude Code 안에서 매주 실행. 벤치마크 기준(`industry-news.md`)만 자사 업종값으로 교체하면 됨.

**아쉬웠던 점 (정직하게)**
- **인과가 아닌 상관까지만**: "메타 과집행이 손실"은 데이터 패턴이지, 왜 그랬는지(소재·타겟 원인)는 이 도구가 답하지 못함.
- **채널 교차 기여 미반영**: 오가닉 매출엔 유료 광고의 낙수효과가 섞여 있는데, 채널을 독립 비교하면 이걸 놓침(MTA/MMM 필요).
- **순증 매출은 상한 추정**: 선형확장 가정이라 실제 효과는 하회 가능.

**다음 스텝 — "2주 더 주어진다면"**
1. 광고 플랫폼 API 직접 연동 → CSV 다운로드조차 없이 완전 자동.
2. 이슈에 원인 진단 한 겹 추가(소재·타겟·랜딩 중 어디가 약한지).
3. 재배분안을 소액 집행 → 실제 성과 재측정 → 다음 배분에 반영하는 **학습 루프**.

*발표 멘트*: "못 한 것도 말씀드립니다. 이 도구는 '무엇이 일어났나'까지고, '왜'는 아직 사람이 봐야 합니다 — 그게 다음 2주에 붙일 부분입니다."

## 슬라이드 7 — 마무리 (4:45–5:00)

> 자기소개·경력 반복 금지. 이 문제에 대한 마무리만.

- **계속 맡는다면**: 지금은 마케팅 성과 리포트지만, 같은 3분할(계산=Python·해석=Claude·반복=Skill) 틀은 **어느 팀의 주간 리포트에도 그대로 확장** 가능 — 사내 리포팅 자동화의 첫 블록.
- **이 문제와 나**: 숫자를 정확히 뽑는 것에서 멈추지 않고, **"틀리지 않는가(재현성)"와 "그래서 뭘 하나(실행안)"까지 스스로 밀어붙이는** 방식으로 풀었습니다. 이번 결과물이 그 근거입니다.

*발표 멘트*: "리포트를 '만드는 일'을 넘어, '틀리지 않고 매주 도는 시스템'으로 만드는 게 제가 이 문제를 본 방식입니다. 감사합니다."

## (별첨) 기술 & 논리 상세

> 깊은 기술 검증을 원하는 담당자만. 본문은 위까지로 완결.

**파이프라인 구조 (`src/`)**
- `calculate.py` — 정제·집계: 숫자형 변환, 완전 중복 제거, 채널별 AOV modified z-score>3.5로 매출 이상치 동적 탐지(revenue만 제외), MER·유료 순효율·오가닉 분리, 채널별 ROI/ROAS/CTR/CVR/CPA.
- `detect_issues.py` — 규칙 기반 이슈 탐지: LOO 중앙값 기준선 + 데이터 적응형 임계값(Tukey IQR 펜스, 하한 25%), 과집행은 ROAS↓ 조건. 손실은 원(₩) 임팩트 정렬, 나머지는 카테고리 분리. 벤치마크 평가·오가닉 기회 탐지.
- `reallocate.py` — `run_pipeline`: 정제→집계→이슈→랭킹→재원→수혜→재배분을 한 번 실행해 결과 dict 반환(md·html 공용). 재원=예산 레버 손실만, 우선순위=임팩트×빈도.
- `generate_report.py` / `generate_html.py` — 같은 `run_pipeline` 결과로 md·HTML 조립(동적 섹션 번호, 수치 불일치 원천 차단).

**재현성 검증 (자동 회귀 테스트)**
- `tests/verify_reproducibility.py` — 원본 + 변형 3종(주차·채널 축소 / 오가닉 제거 / 주차 재라벨)에서 산출물 4개가 크래시 없이 데이터에 맞게 생성되는지 자동 점검(실패 시 exit 1).

**Skill 패키징**
- `.claude/skills/marketing-report/SKILL.md` — "마케팅 성과 CSV 분석" 요청에 자동 발동, 위 파이프라인 오케스트레이션.

**핵심 설계 원칙**
- 계산=Python(정확·재현) / 해석=규칙+Claude / 자동화=Skill. 모든 표현물(md·html)은 단일 계산 결과를 공유 → '데이터 하나, 표현 여럿'.

**의사결정 기록**
- `decisions.md` — 정제·이상치·이슈·벤치마크·재배분 등 모든 판단과 근거(Step 1~). 지표 선택 이유는 `docs/REPORT_DESIGN.md`.

**실행 명령**
```bash
python3 src/generate_report.py [입력CSV] [출력MD]   # 진단 리포트
python3 src/reallocate.py       [입력CSV] [출력MD]   # 예산 재배분안
python3 src/generate_html.py    [입력CSV]            # HTML 대시보드 2개
python3 tests/verify_reproducibility.py             # 재현성 자동 검증
```
