
### T-001 · 마크다운 취소선 오작동 (채널 평가 글씨 지워짐)
**발생 상황**: insight_report.md 채널 평가 섹션을 뷰어로 볼 때 일부 글자가 취소선으로 그어져 읽기 어려움.
**증상**: `평균~상위`, `하위~평균`, `W1~W8`처럼 물결표(~)가 한 줄에 2개 이상 있으면 그 사이 텍스트가 취소선(~~) 처리됨.
**확인된 원인**: GitHub 마크다운은 `~text~`를 취소선으로 렌더링. 등급 라벨·날짜 범위에 쓴 `~`가 짝을 이뤄 오작동.
**조치**: (1) 등급 라벨에서 물결표 제거(`평균~상위`→`평균 이상`, `하위~평균`→`평균 이하(개선여지)`). (2) generate_report.build_report 마지막에 남은 `~`를 엔대시(–)로 자동 치환하는 안전장치 추가.
**재발 방지**: 리포트 생성 시 물결표가 항상 엔대시로 치환되므로 어떤 텍스트를 넣어도 취소선이 생기지 않음.

### T-002 · 기회 섹션 없을 때 KeyError 위험
**발생 상황**: detect_opportunity에 매출 비중 조건(min_share) 추가 후, 조건 미달로 기회가 0건이면 리포트 생성 경로 점검.
**증상**: (잠재) 'opportunity' 키가 없을 때 _opportunity가 ranked['opportunity']를 직접 접근해 KeyError 가능. 현재 데이터는 오가닉이 항상 걸려 표면화 안 됨.
**확인된 원인**: 섹션을 무조건 렌더링하는 구조 + 고정 섹션 번호.
**조치**: build_report를 동적 섹션 번호(리스트+enumerate)로 리팩터. 기회 섹션은 'opportunity' in ranked일 때만 추가.
**재발 방지**: min_share=0.99 테스트로 기회 0건 시 크래시 없이 섹션 생략·번호 연속(1~7) 확인.

### T-003 · 손실 이슈 0건일 때 리포트 생성 크래시
**발생 상황**: 재현성 검증 — 주차 4개·채널 3개로 줄인 변형 CSV로 generate_report 실행.
**증상**: `AttributeError: 'NoneType' object has no attribute 'head'` — _issues에서 loss.head(3) 호출.
**확인된 원인**: 손실 이슈가 0건이면 score_and_rank가 'loss' 키를 안 만들어 ranked.get('loss')=None. _issues가 None에 .head() 호출.
**조치**: _issues에서 loss가 None/empty면 "임계값 넘는 손실 이슈 없음" 문구로 대체.
**재발 방지**: 변형 CSV로 재실행해 크래시 없이 섹션 정상 생성 확인. (샘플 데이터만으론 못 잡는 엣지 — 재현성 테스트의 가치)
**후속(W-004)**: 일회성이던 변형 CSV를 tests/fixtures/에 영구 저장하고 tests/verify_reproducibility.py로 자동 회귀 테스트화. 원본+변형 4종을 한 명령으로 상시 재검증 → 이 엣지가 다시 깨지면 즉시 exit 1로 잡힘.
