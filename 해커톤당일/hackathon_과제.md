# 해커톤 과제: 마케팅 성과 리포트 자동화

> **용어 빠른 이해** — 아래 4개 지표를 알면 과제가 훨씬 쉽게 읽힙니다. raw CSV에는 이 값들이 **들어있지 않으며, 직접 계산하는 것이 과제**입니다.
>
> - **CTR** (클릭률): `clicks / impressions × 100` — 광고를 본 사람 중 클릭한 비율
> - **CVR** (전환율): `conversions / clicks × 100` — 클릭한 사람 중 구매·신청·가입한 비율
> - **ROI** (투자수익률): 본 과제 정의(순위 비교용)는 `revenue / spend × 100` — 매출이 광고비의 몇 %인지. 광고비(spend)가 0인 오가닉 채널은 **측정 불가**.
> - **ROAS** (광고수익률): `revenue / spend` — 광고비 1원당 발생 매출

---

## ① 기업 소개

이 과제에는 특정 채용 기업이 없습니다. **데이터 분석가(DA)가 어느 회사에서나 반복적으로 마주치는 직무 공통 과제** — 마케팅 성과 리포팅 자동화 — 를 다룹니다.

당신은 한 이커머스/서비스 기업의 마케팅 데이터 분석가입니다. 회사는 8주간 5개 채널(네이버광고·메타광고·카카오광고·오가닉·이메일)에 마케팅을 집행했고, 일별 성과 데이터가 CSV로 수집되어 있습니다. 마케팅팀과 경영진은 매주·매월 "어느 채널이 효율적인가, 무엇이 변했는가"를 보고받길 원합니다. 채널별 ROI는 예산 재배분의 근거가 되고, 주차별 급변은 이슈 탐지의 시작점이 되며, 전주 대비 변화는 의사결정 속도를 좌우하는 핵심 보고 항목입니다.

---

## ② 현재 페인포인트

DA는 매주·매월 마케팅팀과 경영진을 위한 성과 리포트를 작성합니다. 그런데 그 과정이 매번 **데이터를 뽑고 → 계산하고 → 시각화하고 → 요약하는 수작업의 반복**입니다.

**[가상 시나리오]** 현실의 DA는 매 주기 같은 형태의 리포트를 손으로 다시 만드느라 반나절 이상을 씁니다. Excel에서 ROI·변화율·채널 비교를 수동 계산하다 수식·셀 참조 오류가 발생하고, 결과물은 "수치 나열"에 그쳐 "그래서 무엇을 해야 하는가"를 담지 못합니다. 핵심은 **계산은 정확하게(Python), 해석은 풍부하게(Claude)** 역할을 나누는 것입니다.

- **수작업 반복**: 같은 형태의 리포트를 매 주기 손으로 다시 작성 (반나절 이상 소요)
- **계산 오류 위험**: ROI·변화율·채널 비교를 Excel에서 수동 계산하다 수식·셀 참조 오류 발생
- **인사이트 부재**: 리포트가 "수치 나열"에 그치고, "그래서 무엇을 해야 하는가"를 담지 못함

## ③ 해결 목표

Claude Code와 대화하며 마케팅 성과 데이터를 다루는 과제입니다. 아래 **3단계(Basic → Standard → Challenge)** 중 본인 수준에 맞는 목표를 골라 도전하세요. **위 단계는 아래 단계를 포함하는 누적형**이며, 더 높은 단계일수록 "1회성 리포트"가 아니라 "매주 돌아가는 자동화", 나아가 "데이터→전략 기획"으로 발전합니다. 별도 API 키나 추가 비용 없이 Claude Code 대화 안에서 완료할 수 있습니다.

### 🟢 Basic — 1회성 인사이트 리포트 만들기 (분석가 수준)

제공된 `marketing_performance.csv`(280행 raw)를 정제한 뒤, **Python으로 합계·ROI·변화율을 계산**하고 ① 핵심 수치 요약(총지출·총매출·전체 ROI·총전환수) ② 채널별 ROI 순위표 ③ 비즈니스 영향이 큰 이슈 3가지 ④ 전주 대비 변화율(W7→W8)을 담은 **인사이트 리포트 1개**를 노션/구글 독스 또는 `insight_report.md`로 제출합니다.

*핵심 역량: 데이터 정제 + 정확한 계산 + 비즈니스 해석*

### 🟡 Standard — 매주 돌릴 수 있는 리포트 자동화 (PM 수준)

Basic을 1회성 작업이 아니라 **재사용 가능한 파이프라인**으로 만듭니다. 새 성과 CSV를 넣으면 `정제 → 계산 → 해석 → 리포트`까지 동일 품질로 자동 생성되도록 Claude Code 슬래시 커맨드·스킬 또는 스크립트로 구성합니다. 주간 리포트의 섹션·지표·형식도 본인이 직접 설계합니다.

*핵심 역량: 워크플로 설계 + 재현성 + 자동화*

### 🔴 Challenge — 리포트 봇 + 예산 재배분 기획안 (시니어 분석가/PM 수준)

Standard에 더해, 채널 ROI·이슈를 근거로 **예산 재배분 기획안까지 자동 도출**합니다. 각 기획안은 `문제 정의 / 근거 데이터 / 재배분안 / 우선순위(임팩트·빈도)`를 포함합니다. 즉, 데이터 수집·계산부터 의사결정용 전략 기획까지 한 번에 이어지는 봇을 만드는 것이 목표입니다.

*핵심 역량: 인사이트 → 전략 전환 + 멀티스텝 자동화*

### 기본 작업 흐름 (Basic 기준)

```
marketing_performance.csv (280행 raw 마케팅 성과 데이터) 를 Claude Code에 올리기
        ↓
정제(결측·중복·이상치) → Python으로 합계·ROI·변화율 계산 (수식 오류 방지)
        ↓
Claude가 계산 완료 수치로 인사이트 해석·리포트 서술
        ↓
노션/구글 독스 또는 insight_report.md 로 정리 → 공유 링크 제출
```

> Standard·Challenge는 위 흐름을 **매주 자동 반복되는 봇/파이프라인**으로, 나아가 **예산 재배분 전략 도출**까지 확장하는 것이 목표입니다.

## ④ 제공 더미 데이터

- **파일명**: `data/marketing_performance.csv`
- **행수**: 280행 (8주 W1~W8 × 채널 5개 × 7일)
- 모두 raw 지표입니다 — CTR/CVR/ROAS 같은 계산값은 들어있지 않으며, **직접 계산하는 것이 과제**입니다.

| 컬럼명 | 설명 | 단위 / 예시 |
|--------|------|------------|
| `date` | 일자 | 2026-04-01 |
| `channel` | 채널명 | 네이버광고 / 메타광고 / 카카오광고 / 오가닉 / 이메일 |
| `impressions` | 노출수 | 회 |
| `clicks` | 클릭수 | 회 |
| `spend` | 광고비 (오가닉은 0) | 원 |
| `conversions` | 전환수 | 회 |
| `revenue` | 매출 | 원 |
| `week` | 주차 | W1 ~ W8 |

> **데이터 품질 주의** — 실무 데이터처럼 의도적으로 깔끔하지 않습니다. 집계 전 정제가 필요하며, 무엇을 왜 처리했는지 리포트·`decisions.md`에 명시하세요.
>
> - **결측 3건**: impressions 1건(카카오 2026-04-11) / clicks 1건(메타 2026-04-20) / revenue 1건(네이버 2026-05-07) — 서로 다른 컬럼·행. 0으로 채우지 말고 "데이터 없음"으로 명시.
> - **완전 중복 행 1쌍**: 네이버광고 2026-05-15(W7) 행이 두 번 존재 — 집계 전 dedup 필요(그대로 두면 합계가 부풀려짐).
> - **매출 이상치 1건**: 이메일 2026-04-26 매출이 전환수 대비 ~9배(약 3,200만 원)로 급등 — 입력 오류 의심, 제거 또는 별도 표기.
> - **오가닉 spend=0**: ROI = 매출/광고비에서 0으로 나누기 발생 — "측정 불가/별도 표기"로 예외 처리.
> - **의도된 주별 급등/급락**: 메타 W5 광고비 급등(약 3.5배)·카카오 W4 광고비 급락(약 41%) 등 — 이슈·변화율 탐지 연습용.

## ⑤ 기대 Output

- **제출 형식**: 노션 페이지 공유 링크 또는 구글 독스 링크 (또는 `output/insight_report.md`). Standard·Challenge는 파이프라인(슬래시 커맨드/스킬/스크립트)도 함께 제출합니다.

### 🟢 Basic — 산출물 고정 (아래 항목 그대로 제출)

**포함 항목**:

- **핵심 수치 요약**: 분석 기간 전체 총지출·총매출·전체 ROI·총전환수 + 채널별 합계 — 모두 **Python 계산값** 사용
- **채널별 ROI 비교**: 5개 채널 ROI 순위표 (ROI = 매출 / 광고비 × 100) + 순위 해석 (오가닉은 spend=0이므로 측정 불가 표기)
- **이슈 3가지**: 급증·급락·이상치·결측 중 비즈니스 영향이 큰 3가지 — 단순 수치 나열이 아닌 해석 + 근거 수치 + 권장 조치
- **전주 대비 변화율**: 최근 2주(W7→W8) 비교 — 지출·매출·전환수·CTR 각각의 증감율(%)

### 🟡 Standard — 최소 요건 + 본인 설계

- (필수) 재사용 가능한 파이프라인: 새 성과 CSV를 넣으면 `정제→계산→해석→리포트`가 동일 품질로 자동 생성되는 슬래시 커맨드/스킬/스크립트 + 간단한 사용법
- (필수) 위 파이프라인으로 생성한 "주간 마케팅 리포트" 샘플 1부
- (직접 기획) 주간 리포트의 섹션·지표·형식은 "마케팅팀·경영진에게 무엇이 유용한가"를 기준으로 **본인이 설계** → 왜 그렇게 구성했는지 1—2줄 설명 포함

### 🔴 Challenge — Standard + 기획안, 양식도 본인 설계

- (필수) Standard 산출물 전체
- (필수) 채널 ROI·이슈를 근거로 도출한 예산 재배분 기획안 샘플 1—2건 (문제 정의 / 근거 데이터 / 재배분안 / 우선순위)
- (직접 기획) 기획안 양식과 우선순위 판단 기준(임팩트·빈도 등)을 **본인이 설계** → 판단 근거 설명 포함

> Standard·Challenge는 "정해진 양식 채우기"가 아니라 **어떤 Output이 현업에 유용한지 스스로 정의하는 것**까지 과제에 포함됩니다. 또한 합계·ROI·변화율 계산은 **반드시 Python으로** 처리하세요 — Claude에게 수기 계산을 맡기면 수식 오류가 발생할 수 있으며, Claude는 계산 완료 수치로 인사이트 해석만 담당합니다.
